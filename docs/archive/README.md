# Historical Documents Index

This folder tracks architecture and implementation documents that are useful for context but are not the canonical direction.

Canonical direction is in:
- `docs/architecture/MASTER_DIRECTION.md`

## Status Map

| File | Status | Notes |
|------|--------|-------|
| `README.md` | active | Product-level overview and packaging notes. |
| `IMPLEMENTATION_PLAN.md` | historical | Contains path/layout assumptions that no longer match runtime shape in full. |
| `New_implementation_plan.md` | historical | Useful intent, but partially divergent from current implementation details. |
| `AUDIT_REPORT.md` | historical | Contains valuable risk notes; some findings are already resolved. |
| `accounts.MD` | operational | Not an architecture doc; treat separately from planning artifacts. |

## Usage Rule
- New implementation plans should reference canonical architecture docs first.
- Historical docs can be cited for rationale only, not as source of current truth.
